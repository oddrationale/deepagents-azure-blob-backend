# CLAUDE.md

## Project Overview

Azure Blob Storage filesystem backend for [LangChain Deep Agents](https://github.com/langchain-ai/deepagents). Implements the `BackendProtocol` interface so Deep Agents can use Azure Blob Storage as their virtual filesystem.

**Package name:** `deepagents-azure-blob-backend`
**Import:** `from deepagents_azure_blob_backend import AzureBlobBackend, AzureBlobConfig`

## Commands

```bash
# Install dependencies
uv sync --group dev

# Run unit tests (no external services needed)
uv run pytest tests/test_backend_unit.py -v --tb=short

# Run integration tests (requires Azurite on port 10000)
uv run pytest tests/test_backend_integration.py -v --tb=short

# Start Azurite for integration tests
docker run -p 10000:10000 mcr.microsoft.com/azure-storage/azurite azurite-blob --skipApiVersionCheck --blobHost 0.0.0.0

# Lint
uv run ruff check .

# Type check
uv run ty check

# Build (use --no-sources to verify PyPI-compatible build)
uv build --no-sources
```

## Architecture

```
src/deepagents_azure_blob_backend/
├── __init__.py     # Public API: AzureBlobBackend, AzureBlobConfig
├── backend.py      # Main backend class implementing BackendProtocol
├── config.py       # AzureBlobConfig dataclass
├── _path.py        # Path normalization (virtual paths ↔ blob keys)
└── _utils.py       # FileInfo builder helpers
```

- **Async-first design:** All operations are implemented as async methods (`aread`, `awrite`, etc.). Sync wrappers delegate via `asyncio.run()` with a thread fallback for nested event loops.
- **Lazy initialization:** The Azure container client is created on first use, not at construction time.
- **No directory blobs:** Directories are synthesized from blob name prefixes. There are no marker blobs for directories.
- **Metadata:** Blobs store `created_at` and `modified_at` as ISO 8601 strings in blob metadata.
- **Prefix isolation:** Each backend instance can be scoped to a prefix, enabling multi-agent setups on a single container.

## Code Style

- **Build system:** `uv_build` (not setuptools or hatchling)
- **Python:** 3.11+ compatibility required
- **Line length:** 120 characters
- **Linter:** ruff with rules `E`, `F`, `I`, `W`
- **Formatter:** ruff format
- **Type checker:** ty (Astral's type checker, not mypy)
- **Docs:** pdoc with `-d google` (Google-style docstrings)
- **Private modules:** Prefixed with underscore (`_path.py`, `_utils.py`)
- **Public API:** Only `AzureBlobBackend` and `AzureBlobConfig` are exported from `__init__.py`

## Testing

- **Unit tests** (`test_backend_unit.py`): No I/O, test path normalization, config defaults, and protocol conformance.
- **Integration tests** (`test_backend_integration.py`): Require Azurite. Test actual read/write/edit/glob/grep against blob storage.
- **Async mode:** `asyncio_mode = "auto"` — async test functions are detected automatically.
- **Isolation:** Each test gets a unique blob prefix via fixtures, so tests don't interfere with each other.

## Deep Agents Integration

The correct API for creating a Deep Agent with this backend:

```python
from deepagents import create_deep_agent
from deepagents_azure_blob_backend import AzureBlobBackend, AzureBlobConfig

config = AzureBlobConfig(
    account_url="https://<account>.blob.core.windows.net",
    container_name="agent-workspace",
    prefix="session-001/",
)
backend = AzureBlobBackend(config)
agent = create_deep_agent(backend=backend)  # NOT create_agent
```

Key points:
- The function is `create_deep_agent`, **not** `create_agent`.
- `model` defaults to `claude-sonnet-4-6` if not specified.
- `backend` is passed as a keyword argument directly to `create_deep_agent`.
- Always call `await backend.close()` when done.
